* Welcome to Lesson 9!

You've been using writing mode (~F8 F8 m~) throughout the previous lessons, but ~org-scribe~ actually offers *four different modes* for different writing workflows. Each mode transforms your environment in unique ways to support specific tasks.

In this lesson, we'll explore all four modes in detail and learn when to use each one.

By the end of this lesson, you'll have:
- Understood all four writing modes and their purposes
- Learned the specific features each mode provides
- Practiced switching between modes efficiently
- Discovered which mode fits your personal writing style
- Customized mode settings to your preferences

*Estimated time*: 15-20 minutes

* The Four Writing Modes

~org-scribe~ provides four mutually exclusive modes:

** Overview Table

| Mode             | Key  | Purpose                       | Best For                      |
|------------------+------+-------------------------------+-------------------------------|
| Distraction-Free | ~m~    | Clean, focused writing        | Daily writing, first drafts   |
| Focus Mode       | ~f~    | One section at a time         | Intensive scene work          |
| Project Mode     | ~p~    | Navigation + writing          | Complex projects, jumping     |
| Editing Mode     | ~e~    | Three-pane editing layout     | Revision, notes integration   |

** Mutual Exclusivity

*Important*: Only one mode can be active at a time!

When you activate a mode:
1. Any other active mode automatically deactivates
2. Previous settings are restored before the new mode activates
3. This prevents conflicts and maintains clean state

*What this means*:
- ~F8 F8 m~ when Focus Mode is active â†’ Focus exits, Distraction-Free activates
- ~F8 F8 p~ when Editing Mode is active â†’ Editing exits, Project activates
- Activating the same mode again â†’ Exits that mode (toggle behavior)

** Quick Summary

Before we dive deep, here's the essence of each mode:

1. *Distraction-Free* (~m~): Clean canvas, no distractions, just you and words
2. *Focus* (~f~): Like Distraction-Free, but shows only current section
3. *Project* (~p~): Two navigation panels + writing area (overview + detail)
4. *Editing* (~e~): Three panels - outline + manuscript + notes (serious revision)

* Mode 1: Distraction-Free Writing Mode

** What It Does

Activates: ~F8 F8 m~

*Visual changes*:
- Switches to clean, minimal theme (default: ~poet~)
- Changes font to writing preset (larger, more readable)
- Enables ~writeroom-mode~ (centers text, hides mode line)
- Shows line numbers
- Narrows text width to comfortable reading (default: 80 columns)

*The result*: A clean, centered, distraction-free writing canvas

** When to Use

Perfect for:
- Daily writing sessions (your default mode)
- First draft writing (no distractions needed)
- Getting into flow state
- Timed writing sprints
- When you want to focus purely on words

*Not ideal for*:
- Navigation between scenes (no outline visible)
- Reference checking (can't see notes)
- Structural editing (need broader view)

** How It Works

1. *Before activation*: Normal Emacs with your usual theme, windows, etc.
2. *Press* ~F8 F8 m~
3. *Screen changes*:
   - Theme switches (dark/complex â†’ light/minimal)
   - Font increases (easier reading)
   - Text centers on screen
   - Mode line simplifies
   - Everything but your text fades away
4. *Write freely!*
5. *Press* ~F8 F8 m~ again to exit
6. *Everything restores*: Theme, font, windows - all back to normal

* Mode 2: Focus Mode

** What It Does

Activates: ~F8 F8 f~

*Everything from Distraction-Free mode, PLUS*:
- Narrows buffer to current org section
- Shows *only* the scene/chapter at point
- Everything else disappears temporarily

*The result*: Laser focus on a single section

** When to Use

Perfect for:
- Intensive work on one scene
- Rewriting a difficult section
- When rest of manuscript is distracting
- Final polish on specific scenes
- Getting unstuck on a problem section

*Not ideal for*:
- First drafts (you might want to see context)
- Jumping between scenes (must widen, navigate, narrow again)
- Checking continuity (need to see other scenes)

** The Difference from Distraction-Free

| Aspect         | Distraction-Free      | Focus Mode                 |
|----------------+-----------------------+----------------------------|
| *Visibility*     | Entire manuscript     | Current section only       |
| *Scrolling*      | Can scroll anywhere   | Scrolls within section     |
| *Navigation*     | All headings visible  | Only current heading       |
| *Mental Model*   | "Work on manuscript"  | "Work on THIS scene"       |
| *When to Exit*   | After writing session | After finishing section    |

** How It Works

1. *Position cursor* on the scene/chapter you want to focus on
2. *Press* ~F8 F8 f~
3. *Buffer narrows*:
   - Only current subtree visible
   - Everything else hidden (not deleted!)
   - Scrolling limited to visible content
   - Plus all Distraction-Free mode effects
4. *Work on that section*
5. *Press* ~F8 F8 f~ again to exit
6. *Buffer widens*: Full manuscript visible again + normal environment restored

* Mode 3: Project Mode

** What It Does

Activates: ~F8 F8 p~

*Layout changes*:
- Opens ~treemacs~ on the left (project file tree)
- Opens ~imenu-list~ on the right (current file outline)
- Main buffer in center (your manuscript)
- Three-panel layout for navigation + writing

*The result*: Overview of project structure + document outline + writing area

** When to Use

Perfect for:
- Complex projects with many files
- Jumping between planning files and manuscript
- Navigating large manuscripts quickly
- When you need structural overview
- Managing multiple chapters/sections

*Not ideal for*:
- Pure writing (panels are distracting)
- Small screens (too cramped)
- Short stories (unnecessary complexity)

** The Two Navigation Panels

*** Left Panel: Treemacs (Project Files)

*Shows*:
- All files in your project
- Directory structure
- Icons for file types

*Use for*:
- Opening character profiles
- Switching to plot document
- Accessing notes files
- Managing project files

*Actions*:
- Click file to open
- Expand/collapse directories
- See project at a glance

*** Right Panel: Imenu-List (Document Outline)

*Shows*:
- Headings in current file
- Hierarchical structure
- Acts â†’ Chapters â†’ Scenes

*Use for*:
- Jumping to specific scene
- Seeing manuscript structure
- Quick navigation within file

*Actions*:
- Click heading to jump
- See where you are in story
- Navigate large documents

** How It Works

1. *Press* ~F8 F8 p~
2. *Frame splits into three*:
   - Treemacs opens (left)
   - Imenu-list opens (right)
   - Current buffer stays (center)
   - Focus remains in center (ready to write)
3. *Use navigation panels as needed*:
   - Click treemacs to open files
   - Click imenu-list to jump within file
   - Keep working in center panel
4. *Press* ~F8 F8 p~ again to exit
5. *Panels close*: Back to normal single-buffer view

** Workflow Example

*Scenario*: Writing a scene, need to check character details.

1. ~F8 F8 p~ (activate Project mode)
2. Left panel (treemacs) â†’ click ~plan/characters.org~
3. Character file opens in center
4. Read character details
5. Left panel â†’ click ~novel.org~
6. Manuscript returns to center
7. Continue writing with fresh details
8. ~F8 F8 p~ (exit) when navigation not needed

* Mode 4: Editing Mode

** What It Does

Activates: ~F8 F8 e~

*Advanced three-panel layout*:
- Left: ~imenu-list~ (document outline - 25% width)
- Center: Current manuscript (50% width)
- Right: Companion notes file (25% width)
- Applies editing-optimized theme and fonts
- Centers text with margins

*The result*: Professional editing environment with notes side-by-side

** When to Use

Perfect for:
- Revision sessions (second draft onward)
- Editing with annotations
- Working with ~org-remark~ notes
- Comparing manuscript to planning notes
- Serious structural editing

*Not ideal for*:
- First draft writing (too complex)
- Small screens (needs width)
- Quick edits (overkill)
- Daily writing sessions (use Distraction-Free instead)

*Purpose*: Your actual editing area

*Use for*:
- Making edits
- Reading scenes

*Purpose*: Companion notes for manuscript

* Common Questions

** "Which mode should I use for daily writing?"

*Answer*: Distraction-Free mode (~F8 F8 m~)

It's the most popular for daily writing because:
- Clean, minimal interface
- Easy to activate/deactivate
- No navigation panels to distract
- Full manuscript visible (can scroll to reference earlier scenes)
- Fast toggle (same key enters and exits)

** "Can I use writing modes outside of org-scribe projects?"

*Answer*: Technically yes, but some features require project structure.

- Distraction-Free (~m~) works anywhere
- Focus (~f~) works in any org-mode file
- Project (~p~) needs a project (any project.el project)
- Editing (~e~) works with any file (creates matching notes file)

Best experience is within a proper writing project!

** "Do modes save my theme/font permanently?"

*Answer*: No, changes are temporary.

- Modes change theme/font when activated
- Original settings are saved
- Exiting mode restores original settings
- Your actual Emacs configuration is unchanged

This is intentional - modes are temporary environments!

** "Why can't I have Project mode AND Distraction-Free at once?"

*Answer*: They conflict in their goals.

- Distraction-Free hides everything â†’ minimal
- Project shows panels â†’ maximal navigation
- These are opposite approaches
- Mutual exclusivity prevents confusion

Choose the right tool for your current task!

** "Can I customize which panels Editing mode shows?"

*Answer*: Currently, the panels are fixed (imenu-list + notes).

But you can:
- Adjust panel widths (percentages)
- Choose different theme
- Change text width
- Use different font

*Future enhancement*: Customizable panel content is planned!

** "What if I don't have treemacs/imenu-list installed?"

*Answer*: Modes gracefully handle missing packages.

- Project mode (~p~): Shows warning, some features missing
- Editing mode (~e~): Works but no imenu-list panel
- Distraction-Free/Focus (~m~/~f~): Require writeroom-mode (show error)

Install recommended packages for full experience!

** "Can I create my own custom mode?"

*Answer*: Advanced users can!

Study existing modes in ~modes/writing-modes.el~:
1. Create a minor mode
2. Add to ~my-writing-exclusive-modes~ list
3. Call ~my-writing--deactivate-other-modes~ when activating
4. Save/restore state appropriately

This is advanced - stick with the four built-in modes initially!

* Troubleshooting

** "Mode activates but looks wrong/broken"

*Check*:
1. Do you have required packages installed?
   - Distraction-Free/Focus need: ~writeroom-mode~
   - Project needs: ~treemacs~ and ~imenu-list~
   - Editing needs: ~imenu-list~

2. Are themes installed?
   - ~M-x customize-themes~ to see available
   - Install missing themes via package manager

3. Is your frame wide enough?
   - Editing mode needs ~120+ columns
   - Project mode needs ~100+ columns
   - Narrow terminal? Modes may look cramped

** "Exiting mode doesn't restore everything"

*Try*:
1. Activate the mode again: ~F8 F8 X~ (where X is the mode key)
2. Exit properly: ~F8 F8 X~ again
3. If still broken: ~M-x revert-buffer~ or close/reopen file

*Prevention*:
- Always exit modes with the same key that activated them
- Don't kill buffers while mode is active
- Don't force-quit Emacs while in a writing mode

** "Focus mode won't narrow / stays wide"

*Causes*:
1. Cursor not on an org heading
2. Not in org-mode buffer
3. Already at top level (can't narrow to whole file)

*Solution*:
- Position cursor on a scene or chapter heading (*** or **)
- Make sure buffer is in ~org-mode~
- Try from a deeper heading (not the document title)

** "Project mode panels are too wide/narrow"

*Solution*:
- Panels size is controlled by the packages themselves
- For imenu-list: ~M-x customize-variable RET imenu-list-size~
- For treemacs: ~M-x customize-variable RET treemacs-width~
- Adjust and restart Project mode

** "Editing mode notes file opens in wrong place"

*Expected behavior*:
- Notes file is always in same directory as manuscript
- Named: ~[manuscript-name]-notes.org~

If opening elsewhere:
- Check current buffer's ~buffer-file-name~
- Notes file derives from this name
- Make sure manuscript is saved to disk (not unnamed buffer)

** "Themes don't change when mode activates"

*Possible causes*:
1. Theme not installed
2. Consult package not available (uses fallback)
3. Custom theme variable not set

*Solutions*:
#+BEGIN_SRC emacs-lisp
;; Check current theme
(custom-enabled-themes)

;; Manually test theme switching
(load-theme 'poet t)

;; Install consult for smoother theme switching
M-x package-install RET consult RET
#+END_SRC
