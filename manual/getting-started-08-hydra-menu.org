By now you've used the Hydra menu many times throughout the previous lessons. You've pressed ~F8 F8~ to insert scenes, count words, search for characters, and more. But you might not fully understand how the menu is organized or what all those options do.

* Before You Begin

** What You Need

✓ A writing project open (any project from previous lessons)
✓ Basic familiarity with the Hydra menu from previous lessons

** Quick Check

Can you open the Hydra menu?

1. Press ~F8~ once
2. Press ~F8~ again
3. You should see the menu appear

If you see the menu, you're ready! If not, make sure ~org-scribe-mode~ is active.

* What is the Hydra Menu?

** The Problem It Solves

~org-scribe~ has dozens of features:
- Insert scenes and chapters
- Activate different writing workspaces
- Capture characters, locations, plots
- Count words and track progress
- Search by character, plot, location, PoV
- Manage ID-based links

*How do you remember all those commands?*

You don't need to! The Hydra menu provides a visual, organized interface to all features.

** Hydra: Sticky Menus

The menu is called a "Hydra" because:
- It shows available options visually
- You press single keys to activate features (no M-x needed)
- Multiple related actions are grouped together
- Submenus are available for complex features
- The menu stays open until you quit or execute a command

*Think of it as a quick-access toolbar* - but keyboard-driven!

* Opening the Hydra Menu

** The Default Keybinding

Press: ~F8 F8~ (F8 twice)

*Why F8 twice?*
- F8 is easy to reach
- Double-tap prevents accidental activation
- Consistent with other Emacs conventions (like C-c C-c)

** What You'll See

When you open the menu, you'll see:

This is your command center!

** Understanding the Layout

The menu is organized into *six columns*:

1. *Insert* - Add content (scenes, chapters)
2. *Workspace* - Change writing environment
3. *Tag Scene* - Link the current scene to characters, locations, plot
4. *Tools* - Word counting and utilities
5. *Search* - Find scenes by properties
6. *Manage* - Capture, links, health, planner

Each column groups related functions together.

* The Six Functional Categories

Let's explore each category in detail.

** Column 1: Insert

*Purpose*: Add new content to your manuscript

| Key | Command      | What It Does                          |
|-----+--------------+---------------------------------------|
| ~s~   | Scene        | Insert a new scene with template      |
| ~c~   | Chapter      | Insert a new chapter with template    |
| ~o~   | Open file    | Quick navigation to project files     |

*When to use*:
- Starting a new scene (~s~)
- Beginning a new chapter (~c~)
- Switching between project files (~o~)

*Most frequent*: ~s~ (scene) - you'll use this constantly!

** Column 2: Workspace

*Purpose*: Change your writing environment for different tasks

| Key | Command     | What It Does                              |
|-----+-------------+-------------------------------------------|
| ~m~   | Write       | Distraction-free writing mode             |
| ~f~   | Focus       | Narrow to current section                 |
| ~e~   | Edit        | Three-panel editing layout                |
| ~v~   | Navigate    | Project navigation panels (treemacs)      |
| ~0~   | Normal      | Return to normal layout                   |

*When to use*:
- Starting writing session → ~m~ (distraction-free)
- Revising one scene → ~f~ (focus mode)
- Editing with notes → ~e~ (editing mode)
- Need outline view → ~v~ (navigation mode)
- Done with any mode → ~0~ (back to normal)

*Most frequent*: ~m~ (write mode) for daily writing

*Tip*: Re-pressing the active mode key (~m~ while in write mode) exits it. ~0~ always returns to normal regardless of which mode is active.

** Column 3: Tag Scene

*Purpose*: Link the current scene to characters, locations, and plot threads using stable ID-based links

| Key | Command         | What It Does                              |
|-----+-----------------+-------------------------------------------|
| ~p~   | PoV             | Set the PoV character for this scene      |
| ~h~   | Characters      | Set all characters present in this scene  |
| ~l~   | Locations       | Set the location(s) for this scene        |
| ~g~   | Plot threads    | Set plot threads active in this scene     |
| ~j~   | Jump to PoV     | Jump to the PoV character's profile       |

*When to use*:
- After writing a scene → tag it so search functions can find it
- Use ~p~ first (PoV), then ~h~ (characters), ~l~ (location), ~g~ (plot)
- Use ~j~ to quickly check who your PoV character is

*Why this matters*: Once scenes are tagged, search keys ~1~–~4~ can find all scenes for any character, location, or plot thread instantly.

** Column 4: Tools

*Purpose*: Writing utilities and reference tools

| Key | Command        | What It Does                                              |
|-----+----------------+-----------------------------------------------------------|
| ~w~   | Count words    | Count words in region or buffer; ~C-u w~ = full update    |
| ~a~   | Update all     | Refresh all heading =WORDCOUNT= properties + sync plan    |
| ~r~   | Track table    | Update progress tracking table (org-tracktable)           |
| ~d~   | Dictionary     | Look up word in RAE dictionary (Spanish)                  |
| ~y~   | Synonyms       | Find synonyms (Spanish/English)                           |
| ~x~   | Thesaurus      | English thesaurus lookup (requires powerthesaurus)        |

*When to use*:
- After writing → ~w~ (check word count quickly)
- After writing → ~a~ (store word counts in headings, sync plan)
- End of session → ~r~ (update progress table)
- Writing in Spanish, need definition → ~d~
- Need a better word → ~y~ (synonyms) or ~x~ (thesaurus)

*Most frequent*: ~w~ (quick count) and ~a~ (full update)

*Key difference between ~w~ and ~a~*:
- ~w~ reports the count; ~a~ updates all =:WORDCOUNT:= properties in the file (and syncs the writing plan if active)
- ~a~ is what you want at the end of a writing session

*Tip*: If you set =(setq org-scribe-auto-wordcount t)= in your init file, ~a~ runs automatically every time you save — no need to press it manually.

*Language note*: Dictionary and synonyms work best for Spanish, but synonyms supports English too

** Column 5: Search

*Purpose*: Find scenes by properties (character, plot, location, PoV)

| Key | Command   | What It Does                             |
|-----+-----------+------------------------------------------|
| ~1~   | by PoV     | Find all scenes from a character's PoV   |
| ~2~   | Character  | Find all scenes featuring a character    |
| ~3~   | Plot       | Find all scenes in a plot thread         |
| ~4~   | Location   | Find all scenes in a location            |
| ~5~   | TODOs      | Find all TODO items in project           |
| ~6~   | Edits      | Find all edit/revision notes in project  |

*When to use*:
- "Where did Alex appear?" → ~2~ (character search)
- "Which scenes have Alex's PoV?" → ~1~ (PoV search)
- "Where's the romance subplot?" → ~3~ (plot search)
- "All coffee shop scenes?" → ~4~ (location search)
- "What needs fixing?" → ~5~ (TODO search)
- "What edits are flagged?" → ~6~ (edit notes)

*Most frequent*: ~2~ (character) and ~3~ (plot)

*Why numbers?*
- Easy to remember: 1=PoV, 2=Character, 3=Plot, 4=Location
- Quick to type
- Consistent ordering

*Note*: Search works best when scenes are tagged (Column 3). Tag first, search later!

** Column 6: Manage

*Purpose*: Capture notes, manage links, run health checks, open the planner

| Key | Command           | What It Does                                  |
|-----+-------------------+-----------------------------------------------|
| ~n~   | Capture note      | Quick note into the project notes file        |
| ~k~   | Capture entity…   | Open capture submenu (character, location…)   |
| ~L~   | Links & upkeep…   | Open linking and relationship submenu         |
| ~R~   | Relink project    | Add IDs, link names, refresh display names    |
| ~H~   | Health report     | Project health: counts, missing props, orphans |
| ~W~   | Planner…          | Open writing plan submenu                     |
| ~q~   | Quit              | Close the Hydra menu                          |

*When to use*:
- Quick thought mid-writing → ~n~ (capture note)
- Adding a new character → ~k h~ (capture entity → character)
- First-time link setup, relationships → ~L~
- Batch cleanup of links → ~R~ (relink entire project)
- Periodic project review → ~H~ (health report)
- Check writing plan → ~W p~

*Capitals = Submenus*:
- ~k~ → Capture submenu
- ~L~ → Links & upkeep submenu
- ~W~ → Planner submenu

* Exploring the Submenus

When you press ~k~, ~L~, or ~W~, you enter a submenu with more options.

** Capture Entity Submenu (F8 F8 k)

#+BEGIN_EXAMPLE
^Capture into project^
_h_: Character   _b_: Object      _g_: Plot thread
_l_: Location    _t_: Timeline    _n_: Note

_q_: Back to main menu            _Q_: Quit
#+END_EXAMPLE

*What it does*:
- ~h~ - Capture a new character with a comprehensive profile template
- ~l~ - Capture a new location/setting
- ~b~ - Capture an important object (MacGuffin, artifact)
- ~t~ - Capture a timeline event
- ~g~ - Capture a plot thread
- ~n~ - Capture a general project note
- ~q~ - Return to main Hydra menu
- ~Q~ - Quit Hydra entirely

*When to use*: Mid-writing, when you invent a new character or place and want to record it without losing your flow.

** Links & Upkeep Submenu (F8 F8 L)

#+BEGIN_EXAMPLE
^Whole project^          ^Set up linking^       ^Relationships^             ^Analysis & Display^
_R_: Relink everything    _c_: Characters        _a_: Add relationship       _t_: Plot health report
_j_: Jump to plot thread  _o_: Locations         _x_: Remove relationship    _S_: Plot statistics
                          _p_: Plot threads      _v_: View relationships     _C_: Column view toggle
                          _r_: Relationships     _V_: View all relationships _T_: Tooltips toggle
                                                 _g_: Insert graph block
_q_: Back to main menu    _Q_: Quit
#+END_EXAMPLE

*What it does*:
- ~R~ - Relink project: the headline command. Runs one pass across the entire project — adds any missing IDs, converts plain character/location/plot names to ID links, and refreshes all display names. This replaces the old per-entity "add IDs / link / update" commands.
- ~j~ - Jump to a plot thread's profile
- ~c~ / ~o~ / ~p~ - One-time setup wizard for character / location / plot linking
- ~r~ - Setup character relationship tracking
- ~a~ / ~x~ - Add or remove a relationship between two characters
- ~v~ / ~V~ - View relationships (one character / all characters)
- ~g~ - Insert a relationship graph dynamic block
- ~t~ / ~S~ - Plot thread health report / statistics
- ~C~ - Toggle clean display in org-column-view (strips =[[id:...]]= syntax)
- ~T~ - Toggle entity tooltips (shows character info in echo area when point is on an ID link)
- ~q~ / ~Q~ - Back to main / Quit

** Planner Submenu (F8 F8 W)

#+BEGIN_EXAMPLE
  org-scribe-planner
  ──────────────────────────────────────────────────
  p: Open project plan (load or create)
  n: New plan
  l: Load plan
  u: Update progress (from word count)
  d: Update daily word count
  r: Recalculate plan

* Common Usage Patterns

Here are the most frequent Hydra workflows you'll use.

** Daily Writing Session

1. ~F8 F8 m~ - Activate writing mode (distraction-free)
2. Write your scene...
3. ~F8 F8 m~ - Exit writing mode (or ~F8 F8 0~)
4. ~F8 F8 w~ - Quick word count check
5. ~F8 F8 a~ - Update all word count properties (syncs plan too)

*Muscle memory*: m → write → m → w → a

** Starting a New Scene

1. ~F8 F8 s~ - Insert scene template
2. ~F8 F8 p~ - Set PoV character (from menu)
3. ~F8 F8 h~ - Set scene characters (multi-select)
4. ~F8 F8 l~ - Set location (from menu)
5. ~F8 F8 g~ - Set plot threads (from menu)
6. ~F8 F8 m~ - Start writing!

*Efficient workflow*: s → p → h → l → g → m

** Tracking Progress

1. ~F8 F8 a~ - Update all heading word count properties
2. Navigate to the word count table
3. ~C-c C-c~ - Refresh the table (regular Org command, not Hydra)
4. ~F8 F8 r~ - Update org-tracktable (if installed)

*Quick check*: a → navigate → C-c C-c

** Finding Character Scenes

1. ~F8 F8 2~ - Search by character
2. Select character from list
3. Review results
4. ~C-c C-o~ - Jump to a scene
5. ~C-c &~ - Return from jump

*Navigation*: 2 → select → C-c C-o → C-c &

** Looking Up a Word (Spanish Writers)

1. Place cursor on word
2. ~F8 F8 d~ - RAE dictionary lookup
3. Read definition
4. OR ~F8 F8 y~ - Find synonyms
5. Choose better word

*Language tools*: d (definition) or y (synonyms)

* Customizing the Hydra Keybinding

The default keybinding is ~F8 F8~, but you can change it.

** Why Customize?

Reasons to change:
- F8 conflicts with other packages
- You want something faster
- You prefer a specific key combination
- Your keyboard layout makes F8 awkward

** Common Alternative Keybindings

Popular choices:

| Keybinding | Pros                            | Cons                         |
|------------+---------------------------------+------------------------------|
| ~F8 F8~      | Default, unlikely to conflict   | Requires two presses         |
| ~C-c w~      | Fast, single combo              | Might conflict with org      |
| ~C-c C-w~    | Similar to other org commands   | Might conflict               |
| ~s-w~        | Super key, unlikely to conflict | Not all keyboards have Super |
| ~M-W~        | Fast, unlikely to conflict      | Might conflict with packages |
| ~F5~         | Single press, easy to reach     | Often used for other things  |

** How to Change the Keybinding

Add to your Emacs configuration (~init.el~ or ~.emacs~):

#+BEGIN_SRC emacs-lisp
;; Example 1: Use C-c w instead
(global-set-key (kbd "C-c w") 'hydra-org-scribe/body)

;; Example 2: Use F5 instead
(global-set-key (kbd "<f5>") 'hydra-org-scribe/body)

;; Example 3: Use Super-w (Windows key + w)
(global-set-key (kbd "s-w") 'hydra-org-scribe/body)

;; Example 4: Use C-c C-w
(global-set-key (kbd "C-c C-w") 'hydra-org-scribe/body)
#+END_SRC

*After adding*:
1. Save your configuration
2. Restart Emacs OR
3. Evaluate the line: ~C-x C-e~ at end of line
4. Test the new keybinding

** Recommended Keybinding

If you're unsure, stick with ~F8 F8~:
- ✓ Unlikely to conflict
- ✓ Easy to remember (double-tap)
- ✓ Default in documentation
- ✓ Consistent for all users

Change only if you have a specific reason!

** Common Mistakes

1. *Forgetting to open Hydra first*
   - Wrong: Just pressing ~m~
   - Right: ~F8 F8~ then ~m~

2. *Using ~w~ when you meant ~a~*
   - ~w~ only reports — it does not update heading properties
   - ~a~ is what actually stores the count and syncs the plan

3. *Confusing ~n~ and ~k~*
   - ~n~ = capture a free-text note
   - ~k~ = open the entity capture submenu (character, location, etc.)

4. *Expecting actions to repeat*
   - Hydra closes after one action (by design)
   - Re-open for next action: ~F8 F8~

5. *Not using submenus*
   - Many powerful features are in ~k~, ~L~, ~W~
   - Don't ignore the Manage column!

** Speed Tips

*Fastest workflows*:
- Scene + writing: ~F8 F8 s ... F8 F8 m~
- Quick count: ~F8 F8 w~
- Full update: ~F8 F8 a~
- Character search: ~F8 F8 2~

*Auto-update shortcut*:
- Add =(setq org-scribe-auto-wordcount t)= to your init file so word counts (and the plan) update silently every time you save — then you never need to press ~a~ manually again.

* Troubleshooting

** "F8 F8 doesn't do anything"

*Possible causes*:
1. ~org-scribe-mode~ not active
   - Check mode line for "Writing"
   - Activate: ~M-x org-scribe-mode~

2. Keybinding not set
   - Check if Hydra package is installed
   - Verify keybinding: ~C-h k F8 F8~

3. Conflict with another package
   - Try ~M-x hydra-org-scribe/body~ directly
   - Set custom keybinding (see Customization section)

** "Menu appears but keys don't work"

*Check*:
1. Are you pressing the right key?
   - Look at the menu carefully
   - Underscores show the exact letter: ~_s_~

2. Is the function available?
   - Some functions need dependencies
   - Example: ~d~ needs the RAE API; ~x~ needs =powerthesaurus=

3. Are you in the right context?
   - Some functions only work in manuscript files
   - Some need cursor on specific heading types

** "Submenu doesn't return to main menu"

*Solution*:
- Press ~q~ in submenu to go back
- Press ~Q~ (capital Q) to quit completely
- If stuck, press ~C-g~ to abort Hydra

** "I pressed the wrong key!"

*Don't panic*:
1. Press ~C-g~ to cancel the command
2. Press ~F8 F8~ to re-open Hydra
3. Press the correct key

*No harm done* - most commands are safe to cancel!

** "Menu looks garbled or misaligned"

*Causes*:
1. Terminal width too narrow
   - Resize Emacs window wider
   - Hydra needs ~100 characters width

2. Font size too large
   - Reduce font size: ~C-x C--~
   - Or use ~C-x C-0~ to reset

3. Theme rendering issue
   - Try different theme
   - Or ignore - functionality still works!

* Summary

The Hydra menu is your command center for ~org-scribe~. Instead of memorizing dozens of command names, you can press ~F8 F8~ and see all available options organized into six clear categories.

The key to mastery is:
1. Understand the six categories
2. Learn your daily workflow keys (s, m, p/h/l/g, w, a)
3. Gradually explore submenus (k, L, W)
4. Practice until ~F8 F8~ becomes muscle memory

Don't try to memorize everything at once. Start with the essentials and expand your knowledge as you need new features!
