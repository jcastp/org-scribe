* Welcome to Lesson 16!

You know how to count words, track progress tables, and log your daily sessions. But do you ever wish Emacs could tell you /how many words you need to write today/ to finish your manuscript on time? That's exactly what this lesson is about.

~org-scribe-planner~ is a planning engine bundled directly inside ~org-scribe~. It handles goal-oriented writing plans: target dates, milestones, daily targets, and progress dashboards. Because it ships with the package, there is nothing extra to install and no integration glue to load — everything wires up automatically the first time you invoke a planner command.

By the end of this lesson, you'll have:
- Created a writing plan for your project from inside Emacs
- Seen your live word count pushed to the plan automatically
- Understood the full planner submenu (~W~) in the Hydra
- Learned how to disable individual features you don't want

*Estimated time*: 20-30 minutes

*Note*: This lesson covers an *optional module*. You can skip it if you prefer to track goals manually.

* Before You Begin

** What You Need

✓ ~org-scribe~ installed and working (you've completed Lessons 1-3)
✓ A project with at least a few scenes and word count properties

That's it. ~org-scribe-planner~ is bundled inside ~org-scribe~ and loads on demand the first time you run any planner command — no separate installation, no ~require~ call in your init file.

** If You Previously Used org-scribe-planner-integration

If your init file contains:

#+BEGIN_SRC emacs-lisp
(with-eval-after-load 'org-scribe-planner
  (require 'org-scribe-planner-integration))
#+END_SRC

you can safely remove that block. The integration features are now built directly into the planner module. The ~org-scribe-planner-integration~ file still exists as a compatibility shim but does nothing — it just prints a reminder message and exits.

* What Does the Integration Do?

Here is an overview of everything the planner provides out of the box. Each feature is independent and can be disabled individually (see [[*Disabling Individual Features][Disabling Individual Features]] at the end of this lesson).

| Feature                        | What happens automatically                                   |
|--------------------------------+--------------------------------------------------------------|
| Auto-load plan                 | Plan file loads when you open any project file               |
| Word count bridge              | Plan reads word counts directly from your manuscript         |
| Auto-update after counting     | ~F8 F8 a~ pushes new total to the plan silently              |
| Project-aware file picker      | Load-plan dialog opens at your project root                  |
| Plan creation on new project   | Offered when you run ~org-scribe-create-novel-project~       |
| Planner Hydra submenu (~W~)    | Full planner menu accessible from the main Hydra             |
| Mode-line status               | Today's target/actual shown as ~[W:actual/target]~           |
| ~org-scribe-plan~ entry point  | Single command that dispatches based on current plan state   |

* Step 1: Create Your Writing Plan

Let's create a plan for an existing project.

** Open Your Project

Open any file inside your org-scribe project (your ~novel.org~, a scene file, etc.).

** Open the Planner Submenu

Press ~F8 F8~ to open the Hydra menu. Press ~W~ to open the planner submenu:

#+BEGIN_EXAMPLE
  org-scribe-planner
  ──────────────────────────────────────────────────
  p: Open project plan (load or create)
  n: New plan
  l: Load plan
  u: Update progress (from word count)
  d: Update daily word count
  r: Recalculate plan
  a: Adjust remaining days
  m: Milestones
  D: Dashboards menu
  q: Back to main menu
#+END_EXAMPLE

** The ~p~ Key — Your Main Entry Point

Press ~p~ (Open project plan) at any time. It does the right thing based on context:

- *Plan already active* → shows the plan calendar immediately
- *Plan file exists but not loaded* → loads it silently, then shows the calendar
- *No plan file found* → asks "No writing plan found for this project. Create one?"
- *Not in an org-scribe project* → falls back to standard load-or-show flow

This means ~F8 F8 W p~ is the one command you need to remember.

** Create a New Plan

If no plan exists, press ~p~ and answer ~yes~, or press ~n~ directly (New plan).

Inside a project, the planner pre-fills defaults from your project metadata — no hunting for paths. It will prompt you for:

1. *Plan file location* — defaults to ~plan.org~ in your project root. Accept or type a custom name.
2. *Total word count target* — e.g. ~80000~ for an 80,000-word novel.
3. *Target completion date* — e.g. ~2026-12-31~.
4. *Daily writing days* — which days of the week you plan to write (e.g. Mon–Fri).

The planner creates ~plan.org~ in your project root and calculates your required daily word count automatically.

*Note*: New projects created with ~org-scribe-create-novel-project~ or ~org-scribe-create-short-story-project~ include a minimal ~plan.org~ placeholder in the project root. You can populate it by pressing ~F8 F8 W p~ right after project creation.

** What the Plan File Looks Like

Open ~plan.org~ to see it:

#+BEGIN_EXAMPLE
* My Novel — Writing Plan
:PROPERTIES:
:TOTAL_WORDS: 80000
:CURRENT_WORDS: 12450
:TARGET_DATE: 2026-12-31
:DAILY_WORDS: 650
:END:
...
#+END_EXAMPLE

The key properties:
- ~TOTAL_WORDS~ — your total manuscript target
- ~CURRENT_WORDS~ — the latest word count (updated automatically)
- ~TARGET_DATE~ — your finish-by date
- ~DAILY_WORDS~ — how many words/day you need to stay on track

* Step 2: Watch Word Counts Sync Automatically

This is where the integration really shines.

** Update Your Manuscript Word Counts

Go to your manuscript (~novel.org~) and run the word count command:

1. Press ~F8 F8 a~ (Add word count properties)
2. Watch the ~:WORDCOUNT:~ properties update on each heading

** Check the Plan

Immediately after step 2, look at your minibuffer. You should see:

#+BEGIN_EXAMPLE
org-scribe-planner: progress updated to 12847 words.
#+END_EXAMPLE

The planner automatically:
1. Summed all ~WORDCOUNT~ properties across your manuscript
2. Wrote the new total to ~plan.org~
3. Saved the plan silently

No extra steps needed. Every time you run ~F8 F8 a~, your plan stays in sync.

** Check the Mode Line

Look at the right end of your mode line. You should see something like:

#+BEGIN_EXAMPLE
[W:450/650]
#+END_EXAMPLE

This means:
- ~450~ — words written today (from your daily log entry)
- ~650~ — today's target to stay on track

Green days are satisfying. Make it a habit!

* Step 3: View Your Plan Calendar

Press ~F8 F8 W p~ (Open project plan).

The planner opens a visual calendar showing:
- Each writing day with its target word count
- Actual words written per day (once you log them)
- Color indicators for ahead / on track / behind
- Milestone markers at key word count thresholds

This gives you an at-a-glance picture of your entire project arc.

* Step 4: Log Your Daily Session

After a writing session, update the daily count.

Press ~F8 F8 W d~ (Update daily word count).

The planner prompts for:
1. The date (defaults to today)
2. How many words you wrote this session

It records the entry and the mode line updates immediately.

*Tip*: Combine this with your existing daily writing log (from Lesson 3). The plan calendar gives you the goal-oriented view; your manuscript's progress table gives you the structural view. Both are valuable!

* Step 5: Keep the Plan Healthy

As you write, things change — you get ahead, fall behind, or adjust your target. Here are the tools for that.

** Recalculate the Plan

Press ~F8 F8 W r~ (Recalculate).

Run this whenever you want to recompute daily targets based on current progress and remaining days. Useful after a productive week (the target goes down!) or when you've been away for a while.

** Adjust Remaining Days

Press ~F8 F8 W a~ (Adjust remaining days).

Use this when your schedule changes — vacation, deadline moved, days off. The planner redistributes the remaining word count across the new schedule.

** Update Progress Manually

Press ~F8 F8 W u~ (Update progress).

This reads the current total directly from your manuscript (same as the automatic sync after ~F8 F8 a~), but you can trigger it any time without re-running the full word count command.

** Milestones

Press ~F8 F8 W m~ (Milestones).

Milestones are named word count checkpoints — "Act I complete", "Halfway!", "First draft done". The planner lets you define them and marks them on the calendar.

** Dashboards

Press ~F8 F8 W D~ (Dashboards menu).

The planner provides various progress dashboards: simple totals, daily averages, streak tracking, burndown charts, velocity trends, and projected completion dates based on your recent pace.

* Step 6: Plan Creation on New Projects

Whenever you create a new project with ~M-x org-scribe-create-novel-project~ or ~M-x org-scribe-create-short-story-project~, the planner will offer:

#+BEGIN_EXAMPLE
Create a writing plan for "My New Novel" with org-scribe-planner? (yes or no)
#+END_EXAMPLE

Answer ~yes~ and the planner opens immediately, ready to set up the new project's plan. The project template also places an empty ~plan.org~ placeholder in the root so that auto-load can find it even before you fill it in.

* How Auto-Loading Works

When you open any file inside an org-scribe project, the planner looks for a plan file automatically. It checks in this order:

0. *Cached path* — if ~.org-scribe-project~ contains a ~# Plan:~ line written during a previous load, that path is used directly (fastest path, skips the scan entirely)
1. ~<project-root>/plan.org~
2. ~<project-root>/<project-title>-plan.org~ (title read from ~.org-scribe-project~)
3. Any ~.org~ file in the project root that contains a ~TOTAL_WORDS~ property

Once a plan file is found via steps 1-3, the path is written back to ~.org-scribe-project~ so future lookups use step 0.

If a matching file is found, the plan loads silently and you see:

#+BEGIN_EXAMPLE
org-scribe-planner: loaded plan "My Novel".
#+END_EXAMPLE

If no plan file exists yet, nothing happens — no error, no interruption.

* Disabling Individual Features

Every integration feature can be turned off independently. Set these variables *before* the planner first loads — the safest place is ~early-init.el~ or anywhere in your init file before the first planner command runs:

** Disable Auto-Loading Plans

#+BEGIN_SRC emacs-lisp
;; Don't load the plan automatically when opening a project file
(setq org-scribe-planner-auto-load-plan nil)
#+END_SRC

** Disable Automatic Word Count Push

#+BEGIN_SRC emacs-lisp
;; Don't push word counts to the plan after F8 F8 a
(setq org-scribe-planner-auto-push-wordcount nil)
#+END_SRC

** Disable Plan Creation Offer on New Projects

#+BEGIN_SRC emacs-lisp
;; Don't offer to create a plan after creating a novel/short story project
(setq org-scribe-planner-offer-plan-on-create nil)
#+END_SRC

** Disable Mode-Line Status

#+BEGIN_SRC emacs-lisp
;; Don't show [W:actual/target] in the mode line
(setq org-scribe-planner-show-mode-line nil)
#+END_SRC

All four can be combined in one block:

#+BEGIN_SRC emacs-lisp
;; Opt out of all planner integration features
(setq org-scribe-planner-auto-load-plan nil
      org-scribe-planner-auto-push-wordcount nil
      org-scribe-planner-offer-plan-on-create nil
      org-scribe-planner-show-mode-line nil)
#+END_SRC

* Practical Exercise

Let's do a complete write-and-track cycle using the planner.

** Exercise: Your First Integrated Session

1. *Open your project* — open ~novel.org~ in your project.
2. *Check auto-load* — after a moment, does the minibuffer show ~org-scribe-planner: loaded plan~? If no plan exists yet, continue to step 3.
3. *Open the planner* — press ~F8 F8 W p~. If no plan exists, answer ~yes~ to create one and follow the prompts.
4. *Write something* — activate writing mode (~F8 F8 m~), write 200 words, exit mode, save (~C-x C-s~).
5. *Update word counts* — press ~F8 F8 a~. Watch the minibuffer confirm the plan updated.
6. *Check mode line* — is ~[W:actual/target]~ visible? Does the actual number match what you just wrote?
7. *Log the session* — press ~F8 F8 W d~ and enter today's session word count.
8. *View the calendar* — press ~F8 F8 W p~ to see your plan.

Congratulations — you have a fully integrated writing tracking system!

* Common Questions

** "The W key doesn't appear in my Hydra menu hint."

This is expected. The ~W~ binding is registered in the Hydra keymap but not shown in the hint (to keep the menu compact). Just press ~W~ — it works even though it's not listed.

** "Auto-load says 'loaded plan' but I didn't create one yet."

The planner found an existing ~.org~ file in your project root that has a ~TOTAL_WORDS~ property (step 3 of the search order). This may be the placeholder ~plan.org~ created by the project template. Open it and fill in your targets, or rename/delete it if you don't want to use the planner yet.

** "The mode line shows [W:0/650] even though I wrote today."

The daily entry doesn't come from the automatic word count sync — it comes from what you log via ~F8 F8 W d~. Run that command and enter how many words you wrote this session. The mode line will update immediately.

** "I want to track words for multiple projects independently."

Each project has its own ~plan.org~ at its root. When you switch between projects (open a file in a different project), the planner detects the new project root and loads that project's plan. The two plans remain separate.

** "Can I use this with a short story project?"

Yes! The planner works identically with short story projects. The plan file goes in the short story's project root (~story.org~'s parent directory).

** "I was using org-scribe-planner as a separate package from Codeberg."

The standalone Codeberg package is superseded by the bundled version. If you still have the standalone package on your load path, the bundled ~planning/org-scribe-planner.el~ takes precedence when loaded through ~org-scribe~. You can safely remove the standalone package from your load path and init file — all functionality is preserved in the bundled version.

* Troubleshooting

** "W in the Hydra does nothing."

- Confirm the planner has been loaded at least once: ~M-: (featurep 'org-scribe-planner) RET~. If it returns ~nil~, trigger a load by running ~M-x org-scribe-plan~.
- Confirm hydra is installed: ~M-: (featurep 'hydra) RET~.

** "Plan doesn't auto-load even though plan.org exists."

- Confirm you're inside an org-scribe project (the directory must have a ~.org-scribe-project~ marker file).
- Confirm ~org-scribe-mode~ is active in the buffer (check mode line for ~Write~).
- Confirm auto-load is enabled: ~M-: org-scribe-planner-auto-load-plan RET~ should return ~t~.
- Try loading manually: ~M-x org-scribe-planner-load-plan~.

** "Word counts don't push to the plan after F8 F8 a."

- A plan must be active. Check: ~M-: org-scribe-planner--current-plan RET~ — should be non-nil.
- Confirm auto-push is enabled: ~M-: org-scribe-planner-auto-push-wordcount RET~ should return ~t~.
- Run ~F8 F8 W u~ (Update progress) manually to test the word count bridge independently.

** "I get a message about org-scribe-planner-integration being deprecated."

Your init file still contains ~(require 'org-scribe-planner-integration)~. Remove that line — the integration is built in and the shim will be removed in a future release.
